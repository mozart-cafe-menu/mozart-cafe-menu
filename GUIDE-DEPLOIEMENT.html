1
Créer un compte GitHub et uploader les fichiers
À faire toi-même
  • 1️⃣
    Aller sur github.com et créer un compte
    Utiliser ton nouveau compte Gmail
  • 2️⃣
    Cliquer sur "New repository" (bouton vert)
    Nom suggéré : mozart-cafe-menu — choisir "Public"
  • 3️⃣
    Uploader TOUS les fichiers du dossier Menu Mozart Café
    Cliquer "uploading an existing file" puis glisser-déposer tout le dossier
💡 Sur GitHub, après avoir créé le repo, clique sur "uploading an existing file" dans la page du repo vide
⚠️
Upload tous les fichiers incluant le dossier api/notify.js — GitHub comprend les sous-dossiers automatiquement
2
Créer le projet Firebase et récupérer les clés
À faire toi-même
🔥 Ouvrir Firebase Console
  • 1️⃣
    Cliquer "Ajouter un projet" / "Add project"
    Nom : mozart-cafe (ou ce que tu veux)
  • 2️⃣
    Dans le projet → cliquer l'icône "</>" (Web app)
    Donner un nom : "Mozart Café Menu", puis cliquer "Enregistrer"
  • 3️⃣
    Copier les valeurs de configuration affichées
    Tu verras un bloc de code avec apiKey, projectId, etc. — garder cette page ouverte
📋 Ce que tu verras sur Firebase (exemple)
const firebaseConfig = { apiKey: "AIzaSyXXXXXXXXXXXXXXXXXXXXXXX", authDomain: "mozart-cafe-xxxxx.firebaseapp.com", projectId: "mozart-cafe-xxxxx", storageBucket: "mozart-cafe-xxxxx.firebasestorage.app", messagingSenderId: "123456789012", appId: "1:123456789012:web:abcdef123456" };
  • 4️⃣
    Activer Realtime Database
    Menu gauche → "Build" → "Realtime Database" → "Create Database" → choisir europe-west1 → "Start in test mode"
  • 5️⃣
    Copier l'URL de la base de données
    Ressemble à : https://mozart-cafe-xxxxx-default-rtdb.europe-west1.firebasedatabase.app
  • 6️⃣
    Obtenir la clé VAPID (pour les notifications)
    Icône roue dentée → "Project Settings" → onglet "Cloud Messaging" → "Web Push certificates" → "Generate key pair" → Copier
  • 7️⃣
    Télécharger le fichier compte de service
    Project Settings → onglet "Service accounts" → "Generate new private key" → Télécharger le fichier JSON
3
Remplir le fichier de configuration
À faire toi-même 1 seul fichier !
✅
Bonne nouvelle : Tu n'as qu'un seul fichier à modifier : firebase-config.js
  • 1️⃣
    Ouvrir le fichier firebase-config.js
    Il est dans le dossier "Menu Mozart Café". Ouvrir avec Notepad ou VS Code.
  • 2️⃣
    Remplacer chaque valeur par tes vraies clés Firebase
    Copier-coller depuis la page Firebase ouverte à l'étape 2
📝 firebase-config.js — Avant
const FIREBASE_CONFIG = { apiKey: "VOTRE_API_KEY", authDomain: "VOTRE_PROJECT_ID.firebaseapp.com", projectId: "VOTRE_PROJECT_ID", storageBucket: "VOTRE_PROJECT_ID.firebasestorage.app", messagingSenderId: "VOTRE_SENDER_ID", appId: "VOTRE_APP_ID", databaseURL: "https://VOTRE_PROJECT_ID-default-rtdb..." }; const VAPID_KEY = "VOTRE_VAPID_KEY";
✅ firebase-config.js — Après (exemple)
const FIREBASE_CONFIG = { apiKey: "AIzaSyXXXXXXXXXXXXXXXXXX", authDomain: "mozart-cafe-xxxxx.firebaseapp.com", projectId: "mozart-cafe-xxxxx", storageBucket: "mozart-cafe-xxxxx.firebasestorage.app", messagingSenderId: "123456789012", appId: "1:123456789012:web:abcdef", databaseURL: "https://mozart-cafe-xxxxx-default-rtdb.europe-west1.firebasedatabase.app" }; const VAPID_KEY = "BxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxVAPIDKEY";
  • 3️⃣
    Mettre à jour le fichier manifest.json
    Remplacer "gcm_sender_id": "YOUR_MESSAGING_SENDER_ID" par ton vrai messagingSenderId
  • 4️⃣
    Ré-uploader ces 2 fichiers modifiés sur GitHub
    Aller sur ton repo GitHub → cliquer sur le fichier → icône crayon → coller → "Commit changes"
💡 Astuce : Pour modifier un fichier sur GitHub, va sur le fichier, clique le crayon ✏️ en haut à droite, modifie, puis clique "Commit changes" en bas.
4
Déployer sur Vercel (hébergement gratuit)
À faire toi-même
▲ Ouvrir Vercel
  • 1️⃣
    S'inscrire sur vercel.com avec ton compte GitHub
    Cliquer "Sign Up" → "Continue with GitHub"
  • 2️⃣
    Cliquer "New Project" → Importer ton repo GitHub
    Chercher "mozart-cafe-menu" dans la liste et cliquer "Import"
  • 3️⃣
    AVANT de déployer : ajouter la variable d'environnement
    Cliquer "Environment Variables" et ajouter :
🔑 Variable d'environnement à ajouter sur Vercel
Name : FIREBASE_SERVICE_ACCOUNT Value : [Coller ICI tout le contenu du fichier JSON téléchargé à l'étape 2.7] Le contenu JSON ressemble à : { "type": "service_account", "project_id": "mozart-cafe-xxxxx", "private_key_id": "...", "private_key": "-----BEGIN RSA PRIVATE KEY-----\n...", "client_email": "firebase-adminsdk-xxxxx@mozart-cafe-xxxxx.iam.gserviceaccount.com", ... }
  • 4️⃣
    Cliquer "Deploy" et attendre ~1 minute
    Vercel va construire et déployer automatiquement
  • 5️⃣
    Copier l'URL de ton site (ex: mozart-cafe-menu.vercel.app)
    C'est l'URL que tu donneras aux clients et aux serveurs !
⚠️
Le fichier JSON du compte de service est secret — ne le partage jamais et ne le mets pas sur GitHub. Il va uniquement dans la variable Vercel.
5
Installer l'app sur les téléphones des serveurs
À faire toi-même
  • 1️⃣
    Sur le téléphone Android du serveur : ouvrir Chrome
    Aller sur https://ton-site.vercel.app/app.html
  • 2️⃣
    Menu Chrome (3 points) → "Ajouter à l'écran d'accueil"
    L'app s'installe comme une vraie application
  • 3️⃣
    Accepter les notifications quand demandé
    Obligatoire pour recevoir les alertes quand l'app est fermée
✅
C'est tout ! Le serveur recevra une notification avec le numéro de table à chaque appel client.
🤖 Option APK (si tu veux un vrai fichier .apk à installer)
📱 Ouvrir PWABuilder
Entrer l'URL de ton site Vercel → "Start" → "Android" → Télécharger le .apk
